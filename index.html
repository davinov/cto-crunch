The platform.sh architecture

Shared nothing architecture, no central service.

    • Outer layer handles:
    • HTTP and SSH reverse proxying
    • DoS mitigation
    • SSL termination
    • MongoDB there mostly for legacy reasons (friends don't let friends use mongo in production)
  • Inner most layer Coordinators a distributed Eye Of Mordor kind of thing
  • Inbetween, the grid of containers, hundreds or thousands per host

Every application cluster is self contained

    • Often
    • The Git server
    • A router
    • App server
    • Possibly a couple of others (micro-services!)
    • A cache
    • A Databse
    • A Search Engine

Dense highly available micro-container grid

  • We can run thousands of useful containers on a single server
  • No operation has downtime
  • Everything is distributed and replicated
  • Yeah, we know Docker. But no.
